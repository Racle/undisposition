Advanced: Content-Type Rules

Chrome's Manifest V3 cannot inspect response headers, so Content-Type rewriting is URL-based. Built-in rules handle common image formats (jpg, png, svg, webp, etc.) automatically. Use the sections below to add rules for other file types.

Extension Rules

Rewrite Content-Type for any URL ending in a given extension. One rule per line. Use with caution — these match all sites.

Example: adding yml=text/plain
example.com/config.yml → text/plain (displays inline)
⚠ Matches every .yml URL on every site, even HTML preview pages
Add: yml=text/plain csv=text/plain json=application/json pdf=application/pdf xml=application/xml

URL Pattern Rules

Rewrite Content-Type only for URLs matching a glob pattern. Use =auto to detect the type from the file extension. Much safer than extension rules.

Example: *gitlab.com*/artifacts/raw/*=auto
gitlab.com/.../artifacts/raw/config.yml → text/plain
gitlab.com/.../artifacts/raw/image.png → image/png
gitlab.com/.../artifacts/raw/Makefile → no match (no extension)
example.com/files/config.yml → not matched (different domain)
